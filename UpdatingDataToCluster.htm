Interaction: Application User Data

1. Data Flow Application User Data Is Potentially Interrupted  [State: Mitigation Implemented]  [Priority: High] 

Category:Denial Of Service
Description:An external agent interrupts data flowing across a trust boundary in either direction.
Justification:Elastic search prevents Denial of Service attacks by disabling dynamic scripts, implementing authentication and role-based access control.

2. External Entity ElasticSearch Cluster Potentially Denies Receiving Data  [State: Mitigation Implemented]  [Priority: High] 

Category:Repudiation
Description:ElasticSearch Cluster claims that it did not receive data from a process on the other side of the trust boundary. Consider using logging or auditing to record the source, time, and summary of the received data.
Justification:Elastic search prevents elevation of privilege by implementing role-based access control and input validation from data sources by adding character filters to an analyzer to process and validate the text before it is passed to the tokenizer.

3. Spoofing of the ElasticSearch Cluster External Destination Entity  [State: Mitigation Implemented]  [Priority: High] 

Category:Spoofing
Description:ElasticSearch Cluster may be spoofed by an attacker and this may lead to data being sent to the attacker's target instead of ElasticSearch Cluster. Consider using a standard authentication mechanism to identify the external entity.
Justification:Elastic search prevents elevation of privilege by implementing role-based access control and input validation from data sources by adding character filters to an analyzer to process and validate the text before it is passed to the tokenizer.

5. Data Flow Data Documents Is Potentially Interrupted  [State: Mitigation Implemented]  [Priority: High] 

Category:Denial Of Service
