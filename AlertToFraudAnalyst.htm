1. Potential Excessive Resource Consumption for 1.0 Sending Alert or Data Index  [State: Mitigation Implemented]  [Priority: High] 

Category:Denial Of Service
Description:Does 1.0 Monitoring and Alerting or Data Index take explicit steps to control resource consumption? Resource consumption attacks can be hard to deal with, and there are times that it makes sense to let the OS do the job. Be careful that your resource requests don't deadlock, and that they do timeout.
Justification:ElasticSearch offers disk space thresholds to ensure system stability in case of data flooding and high disk usage.

2. Spoofing of Destination Data Store Data Index  [State: Mitigation Implemented]  [Priority: High] 

Category:Spoofing
Description:Data Index may be spoofed by an attacker and this may lead to data being written to the attacker's target instead of Data Index. Consider using a standard authentication mechanism to identify the destination data store.
Justification:Data indexes are stored on the file system and strict file permissions should be enforced to control access to the data store. Built in validations could even be built-in to verify proper file permissions when the process starts.

3. Weak Access Control for a Resource  [State: Mitigation Implemented]  [Priority: High] 

Category:Information Disclosure
Description:Improper data protection of Data Index can allow an attacker to read information not intended for disclosure. Review authorization settings.
Justification:Data indexes are stored on the file system and strict file permissions should be enforced to control access to the data store. Built in validations could even be built-in to verify proper file permissions when the process starts.
